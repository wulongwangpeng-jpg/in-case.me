## 一、我们在解决什么问题

### 一个被忽视的刚需

中国有 **10.8 亿网民**，平均每人拥有 **20+ 个数字账户**——微信、支付宝、网银、股票、基金、游戏、社交、云盘、订阅服务……

**人没了，账号还在。** 家属不知道有哪些账号、不知道密码、不知道找谁处理。每一个账号背后都是真金白银或重要回忆，但绝大多数人从没整理过。

### 现有的"解决方案"全都不work

| 方案 | 问题 |
|------|------|
| 写在纸上告诉家人 | 纸会丢、会忘、账户一直在变 |
| 用密码管理器 | 家属不知道怎么用、不知道主密码 |
| 找律师立遗嘱 | 贵（几千起）、重财产不重数字资产、年轻人不会去 |
| 平台自身的"遗产联系人" | 只有苹果/Google有，国内平台几乎不支持 |

**结果是：每年大量数字资产永远"冻结"在互联网上——钱取不出、账号消不掉、回忆找不回。**

### 为什么是现在

- 中国第一代网民（70后/80后）正在进入中年，开始认真思考"万一"
- 年轻人（90后/00后）数字资产复杂度远超上一代：自媒体账号、游戏皮肤、数字藏品、知识付费内容
- "数字遗产"概念正在进入公众视野（民法典已明确虚拟财产可继承）
- 但市面上**没有一个普通人用得起的工具**

## 二、产品：万一呢是什么

### 三步完成数字遗产规划

```
① 资产盘点                              ② 心底话                              ③ 设置信使
盘一遍你有哪些数字资产               给重要的人写封信                   指定信任的人，给他们"钥匙"
AI 聊天式引导 + 直接录入             也可写下对信使的嘱托               你安好时他们打不开
                                                                        你失联 90 天后才能提取
```

### 核心功能矩阵

| 模块 | 功能 | 差异化 |
|------|------|--------|
| **资产盘点** | AI 引导式梳理 + 直接录入 + 分类标签 + 处理方式标记 | 覆盖创作者专属字段（粉丝资产、商务合作、内容版权） |
| **心底话** | AI 辅助写信 + 直接书写 + 可写给信使的嘱托 | 不是"遗书"，是温暖的情感传递 |
| **信使系统** | 创建信使凭证 + 细粒度权限（每位信使可见不同密文）+ 双码机制 | 链接+密码缺一不可，安全与便利的平衡 |
| **续期机制** | 90 天静默触发 + 定期续期提醒 + 两档续期（90天/45天） | 把冷冰冰的"死亡倒计时"变成温暖的仪式 |
| **数据灾备** | 一键导出全部加密数据 + JSON 格式 + 离线可恢复 | 不做数据绑架，随时可以走 |

### 技术亮点（投资人可略过，技术DD用）

- **零知识加密**：AES-256-GCM + PBKDF2，加密在浏览器本地完成，服务器只存密文
- **我们自己也打不开用户数据**——这是合规优势，也是信任基础
- **双码信使机制**：人类可读短码（"枫叶-长颈鹿-37"）+ SHA-256 安全令牌
- **纯时间戳访问控制**：没有服务器端逻辑决定谁能看什么，信使提取的唯一条件是用户失联超过阈值

## 三、市场有多大

### 直接可触达市场（TAM → SAM → SOM）

| 层级 | 人群 | 规模 |
|------|------|------|
| **TAM** | 中国网民 | 10.8 亿 |
| **SAM** | 25-50 岁、有数字资产意识的城市互联网用户 | ≈ 2 亿 |
| **SOM** | 愿意为数字遗产规划付费的早期用户（类比记账App渗透率 3-5%） | ≈ 600-1000 万 |

### 细分市场

1. **大学生**（2000万+/年毕业生）：数字原生代，账号多，几乎没有遗嘱意识，但愿意尝试新鲜工具
2. **30-45 岁普通人**：有家庭、有资产、开始认真考虑"万一"，是最核心的付费人群
3. **自由职业/博主/创作者**：数字资产=收入来源，有强烈的传承需求（粉丝资产、内容版权、商务合作）
4. **慢性病患者/高危职业从业者**：刚需人群，但需要更精准的渠道触达

### 对标参考

| 产品 | 模式 | 用户量 | 估值/收入 |
|------|------|--------|-----------|
| 死了吗（中国） | 打卡续期 + 定时发送 | 约 50 万用户 | 未披露 |
| Lantern（美国） | 数字遗产SaaS | 未披露 | 被收购 |
| GoodTrust（美国） | 数字遗产规划 | 10万+ | $2M+ 种子轮 |
| **万一呢** | 零知识 + AI + 信使体系 | 0（即将推广） | — |

**关键差异化：** 万一呢是唯一一个同时做到"零知识加密 + AI辅助梳理 + 信使细粒度权限"的产品。竞品要么安全性不足（明文存储），要么缺乏AI引导降低使用门槛，要么不面向中国市场和中文语境。

## 四、商业模式

### 当前状态
- **完全免费、开源**，验证产品价值和用户需求
- 运营成本极低：Supabase 数据库 + 服务器 + AI API ≈ **¥100-150/月**

### 付费方案（产品市场验证后上线）

| 方案 | 价格 | 内容 |
|------|------|------|
| **免费版** | ¥0 | 3 条心底话 + 1 个信使 |
| **年付** | **¥168/年** | 无限使用 |
| **终身** | **¥498** | 永久使用 |

### 定价逻辑
- ¥168/年在同类中很有竞争力：印象笔记 ¥148/年、Flomo ¥99/年、1Password $36/年(≈¥260)
- 终身方案锚定心理价位，推动年付转化
- 促销策略：限量 50 个终身 ¥198（而非打折走量），制造稀缺感

### 盈亏平衡点
- 年成本 ≈ **¥876-1476**（服务器 + DB + AI API + 域名）
- **8 个年付用户即可覆盖成本**
- 1000 个年付用户 = ¥16.8 万年收入，单人全栈完全可持续

### 扩展收入方向
- **B2B 企业版**：律所/遗嘱服务机构批量采购，白标定制
- **保险合作**：与寿险/意外险产品捆绑，作为增值服务
- **API 授权**：加密 + 信使体系授权给其他数字遗产平台
- **内容/教育**：死亡教育、数字遗产规划课程（但不应过早分散精力）

## 五、竞争壁垒

| 壁垒类型 | 具体内容 |
|------|------|
| **技术壁垒** | 零知识加密架构（AES-256-GCM + PBKDF2 + 客户端加密管线），不是简单套个库就能做对 |
| **产品壁垒** | AI 引导式梳理——精心设计的 prompt chain，非通用 AI 聊天可比 |
| **信任壁垒** | 开源 + 零知识 = 用户可以验证我们真的看不到数据。这是数字遗产领域的核心护城河 |
| **数据壁垒** | 用户一旦建立了完整的资产图谱和信使关系，迁移成本极高 |
| **先发优势** | 中文市场尚无同级别的零知识数字遗产规划工具 |

## 十、为什么现在投我们

1. **市场时机对了**：中国互联网人口结构正在经历代际更替，第一代网民进入遗产规划年龄。数字遗产不再是"以后的事"。
2. **赛道足够大，竞争足够少**：不像 SaaS/AI 工具那样一片红海。数字遗产规划在中国几乎空白。
3. **产品已可用**：不是PPT项目，真实可用的产品已经上线，你可以打开 demo-gamma-ten.vercel.app 亲自体验。
4. **技术壁垒真实存在**：零知识加密 + AI 引导 + 信使体系，不是两周能复制的。
5. **低风险验证期**：单人全栈 + 月成本 ¥150，融资前可以不断完善产品、积累种子用户。投资人进入时已有一定验证数据。
6. **天花板高**：如果"数字遗产规划"成为像"记账"一样的基础需求，市场空间是百亿级的。
